简介

hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github上。

怎样搭建hexo博客

安装Git

作用：把本地的hexo内容提交到github上。
安装Xcode就自带Git，这步可以忽略。

安装Node.js

作用：用来生成静态页面的。
到Node.js官网下载就OK。

安装hexo

Node和Git都安装好后，可执行如下命令安装hexo:

npm install -g hexo

初始化

然后，执行init命令初始化hexo到你指定的目录：

hexo init

我是直接

hexo init Jenko

生成静态页面

cd到你的init目录，执行如下命令，生成静态页面至hexo\public\目录

hexo generate

或

hexo g

本地启动

执行如下命令，启动本地服务，进行文章预览调试。

hexo server

或

hexo s

浏览器输入http://localhost:4000就可以看到效果。

问题

这里可能会出现一些问题导致本地的预览界面出不来

  • 问题1：

    Besides,utilities are separated into a standalone.hexo.util is not
    reachable anymore.

解决方法，执行命令：

npm install hexo-server --save

  • 问题2：
    什么都没有显示或者显示Cannot GET/
    解决方法，由于2.6以后就更新了，我们需要手动配置些东西，执行命令：

    npm install hexo-renderer-ejs --save

    npm install hexo-renderer-stylus --save

    npm install hexo-renderer-marked --save

GitHub

1.首先注册一个GitHub账号。
2.建立与你用户名对应的仓库，仓库名必须为your_username.github.com
3.在Github上添加SSH公钥。
下图为建立仓库：

添加SSH公钥

首先设置你的用户名和密码：

git config --global user.email "这里是你申请Github账号时的邮箱"
git config --global user.name "这里是你申请Github账号时的name"

生成密钥

ssh-keygen -t rsa -C "邮箱"

下图是我生成密钥的路径，注意！添加到Github中的是id_rsa.pub这个公钥。
在终端中，编辑它拷贝下来到Github中ADD SSHKey就OK了。

部署Hexo到Github上

编辑你初始化后文件夹里的_config.yml
找到下面的位置并修改，username为你的Github name。
注意冒号后面带一个半角的空格在输入。

deploy:
type: git
repository: https://github.com/username/username.github.io.git
branch: master
